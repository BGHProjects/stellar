# Stellar Gateway

The Go API gateway is the single entry point for all frontend requests. The frontend exclusively communicates with this service — no microservice is ever called directly from the browser.

## Responsibilities

- **Authentication** — JWT-based registration, login, token refresh, and facial landmark vector storage
- **Voyage search** — queries routes from `system.json`, calculates orbital-influenced journey durations and pricing in real time
- **Booking management** — creates, retrieves, and cancels bookings stored in local JSON files
- **System config** — serves all orbital, route, ship, and pricing configuration to the frontend
- **Proxy** — forwards requests to the AI chatbot, vision, and quantum routing microservices

## Stack

| Concern          | Choice                                                           |
| ---------------- | ---------------------------------------------------------------- |
| Language         | Go 1.23+                                                         |
| Router           | [chi](https://github.com/go-chi/chi)                             |

## Architecture Notes

### Repository Pattern

The `repository/` package abstracts all data access behind a consistent interface. Handlers and services never read or write JSON files directly — they call repository methods. This means the backing store (currently local JSON) can be replaced with PostgreSQL by implementing the same interface in a new package, without touching any handler or service code.

### Orbital Calculations

The `orbital/` package implements the same orbital mechanics formula used by the frontend TypeScript renderer (`frontend/src/lib/orbital.ts`). Both implementations read from the same `config/system.json` values, so the 3D star map and the backend pricing logic always agree on where every body is.

All orbital parameters (radii, periods, eccentricities) live in `system.json`. Changing a planet's orbital period updates voyage durations, pricing, and window ratings automatically — no code changes required.

### Config-Driven Pricing

The full price formula is:

```
total = basePriceCredits
      × orbitalWindowMultiplier
      × routeTypeMultiplier
      × cabinClassMultiplier
      × cryoOptionMultiplier
      × passengerCount (with child discount applied)
      + addOnsTotal
      + portFees
      − loyaltyDiscount
```

Every multiplier is defined in `system.json`. Adjust any value there and pricing recalculates across all endpoints immediately.

## Prerequisites

- Go 1.23 or later
- `swag` CLI for generating/updating Swagger docs (see below)

Install swag:

```bash
go install github.com/swaggo/swag/cmd/swag@latest
```

## Setup

**1. Clone the repo and navigate to the gateway directory:**

```bash
cd gateway
```

**2. Copy the environment file and fill in your values:**

```bash
cp .env.example .env
```

The only required value is `JWT_SECRET`. Generate one with:

```bash
openssl rand -hex 32
```

**3. Install dependencies:**

```bash
go mod tidy
```

**4. Generate Swagger docs:**

```bash
swag init
```

This reads the annotations in `handlers/*.go` and `main.go` and generates the `docs/` directory. Re-run this command whenever you add or change an endpoint.

**5. Run the gateway:**

```bash
go run .
```

The gateway starts on port `8080` by default (override with `PORT` in `.env`).

## Viewing the API Documentation

With the gateway running, open your browser to:

```
http://localhost:8080/swagger/index.html
```

This serves the full interactive Swagger UI. You can:

- Browse all endpoints grouped by tag
- View request/response schemas
- Execute live requests directly from the browser
- Authenticate using the **Authorize** button (paste a JWT from a `/api/auth/login` response)

The raw OpenAPI JSON spec is available at:

```
http://localhost:8080/swagger/doc.json
```

## API Endpoints

All endpoints are prefixed with `/api`.

### Auth

| Method | Path                | Auth | Description                    |
| ------ | ------------------- | ---- | ------------------------------ |
| POST   | `/auth/register`    | No   | Register a new account         |
| POST   | `/auth/login`       | No   | Login and receive tokens       |
| POST   | `/auth/refresh`     | No   | Refresh access token           |
| GET    | `/auth/me`          | Yes  | Get authenticated user profile |
| POST   | `/auth/face-vector` | Yes  | Store facial landmark vector   |

### System (all public)

| Method | Path                    | Description                           |
| ------ | ----------------------- | ------------------------------------- |
| GET    | `/system/bodies`        | All orbital bodies with render data   |
| GET    | `/system/routes`        | All scheduled routes                  |
| GET    | `/system/ships`         | All ship classes                      |
| GET    | `/system/spaceports`    | All spaceports                        |
| GET    | `/system/cabin-classes` | Cabin class definitions               |
| GET    | `/system/cryo-options`  | Cryostasis options                    |
| GET    | `/system/add-ons`       | All bookable add-ons                  |
| GET    | `/system/loyalty-tiers` | Loyalty programme tiers               |
| GET    | `/system/pricing`       | Pricing configuration and multipliers |

### Voyages (all public)

| Method | Path                         | Description                                 |
| ------ | ---------------------------- | ------------------------------------------- |
| GET    | `/voyages/search`            | Search voyages by origin, destination, date |
| GET    | `/voyages/orbital-positions` | Current positions of all bodies             |
| GET    | `/voyages/closest-approach`  | Find optimal travel window                  |
| POST   | `/voyages/price`             | Calculate full price for a configuration    |
| GET    | `/voyages/{id}`              | Get a single voyage by ID                   |

### Schedule (public)

| Method | Path                  | Description                            |
| ------ | --------------------- | -------------------------------------- |
| GET    | `/schedule/{routeId}` | Rolling departure schedule for a route |

### Bookings (auth required)

| Method | Path                    | Description                         |
| ------ | ----------------------- | ----------------------------------- |
| POST   | `/bookings`             | Create a booking                    |
| GET    | `/bookings/me`          | All bookings for authenticated user |
| GET    | `/bookings/{id}`        | Get a single booking                |
| PATCH  | `/bookings/{id}/cancel` | Cancel a booking                    |

### Proxy

| Method | Path                   | Auth | Description                     |
| ------ | ---------------------- | ---- | ------------------------------- |
| POST   | `/chat`                | Yes  | Forward message to AI chatbot   |
| POST   | `/vision/authenticate` | No   | Facial authentication           |
| POST   | `/vision/enrol`        | Yes  | Enrol facial landmark vector    |
| POST   | `/routing/optimise`    | Yes  | Optimise multi-stop route order |
